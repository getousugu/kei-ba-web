# 🏇 けーば Web版

> DiscordBot版「けーば」をブラウザで遊べるマルチプレイヤー競馬ゲーム。  
> WebRTC (PeerJS) によるP2P接続を使用し、サーバーレスでリアルタイム通信を実現。

---

## 🚀 起動方法

```bash
cd keiba-web
npm install
npm run dev
```

ブラウザで `http://localhost:5173/` を開く。

---

## 🎮 ゲームの流れ

```
ログイン → ロビー → レース設定 → 馬券購入 → レース観戦 → 結果
                ↑______________________________(リマッチ)_____↓
```

1. **ログイン**: プレイヤー名を入力、ルーム作成 or 参加
2. **ロビー**: 参加人数・設定を確認、WIN5開催も可能
3. **レース設定** (ホストのみ): 距離・馬場・天候・コース特性・頭数を選択
4. **馬券購入**: 単勝/複勝/馬連/ワイド/馬単/3連複/3連単を購入
5. **WIN5** (開催時のみ): 5レース連続的中を目指し、脱落時にキャッシュアウトするか続行かを選択
6. **レース**: リアルタイムアニメーションでレース観戦
7. **結果**: 払戻金受け取り・リマッチ投票

## 🎲 ゲームシステム詳細

### 馬のレアリティ

| レアリティ | ステータス平均 | 排出率 |
|------------|--------------|--------|
| ⭐ Common | 低 | 多い |
| 💎 Rare | 中 | 普通 |
| 🔮 Epic | 高 | 少ない |
| 👑 Legendary | 非常に高い | 稀 |

### 脚質とスピードカーブ

| 脚質 | 特徴 |
|------|------|
| 逃げ | 序盤強く終盤垂れる |
| 先行 | バランス型、安定 |
| 差し | 後半加速型 |
| 追込 | 直線で爆発的に伸びる |
| **暴れ馬** | **レースごとにランダムなスピードカーブ。毎回予測不能** |

### コース特性（ホストが選択可能）

| 特性 | 効果 |
|------|------|
| 平坦 | 標準。補正なし |
| **坂あり** | **パワーが高い馬に有利 (+4%)** |
| **直線長** | **バーストが高い馬に有利 (+4%)** |
| **コーナー多** | **賢さが高い馬に有利 (+4%)** |

### ペース判定

逃げ馬の頭数と前走配置によりペースが決まり、前半型/後半型の有利不利に影響します。

- **ハイペース**: 逃げ馬4頭以上 → 差し・追込に有利
- **スローペース**: 逃げ馬0頭 → 逃げ・先行に有利
- **ミドルペース**: 標準

### NPCボット

ホストが「NPCの参戦」をONにすると、15種類の個性を持つNPCが自動でベットします。

| NPC | 戦略 |
|-----|------|
| ウマ吉 | バランス型 |
| 大穴ハンター | 高オッズ馬専門 |
| 堅実派ロボ | 1〜3番人気中心 |
| 逃げ馬命 | 逃げ脚質の馬を優先（先行でフォールバック） |
| 一点集中型 | 上位N頭に集中投資 |
| ... | 他10種 |

### 🏆 WIN5 サバイバル

特定の条件下でロビーから開催できる特別モードです。

- **ルール**: 全5レースの勝ち馬をすべて当てる。
- **サバイバル**: レースごとに的中者（サバイバー）のみが次へ進める。
- **キャッシュアウト**: レース終了時に、その時点までの配当を受け取ってリタイア可能。
- **キャリーオーバー**: 全員脱落した場合、プールされた賞金は次回開催へ持ち越されます。
- **賞金**: 最終（5レース目）の的中者で、プールされた全額＋キャリーオーバー分を山分け。

## 🌐 マルチプレイ同期

- **同期モデル**: ホストが `raceStartTime`（Unix絶対時刻）を全員に配布
- **カウントダウン**: 全クライアントが同一の絶対時刻を参照（ドリフトなし）
- **バッファ**: レース開始5秒前にシミュレーションデータを送信して同期を吸収
- **ペア**: PeerJSのP2P接続。ホストがメッセージルーターを兼任

### メッセージタイプ（主要）

| type | 方向 | 内容 |
|------|------|------|
| `phase_start` | Host→Guest | フェーズ遷移 + データ同期 |
| `sync_full_state` | Host→Guest | 新規参加者への完全状態同期 |
| `place_bet` | Guest→Host | ベット送信 |
| `odds_update` | Host→Guest | オッズ変動を全員に配布 |
| `chat` | Both | チャットメッセージ |
| `vote` | Guest→Host | リマッチ投票 |
| `win5_update` | Host→Guest | WIN5の賞金・生存者情報の同期 |
| `win5_cashout` | Guest→Host | ゲストの途中離脱通知 |

## 💾 データ永続化 (IndexedDB)

| テーブル | 内容 |
|----------|------|
| `players` | 所持金・称号・借金状態 |
| `horses` | 永久馬の成績・レーティング |
| `race_history` | レース収支履歴（WIN5除外） |

### 📊 収支推移グラフ (サイドバー)

過去100レース分のデータから日単位で集計し、ASCIIグラフで表示します。

- **右側 (█)**: その日の純利益（プラス収支）
- **左側 (░)**: その日の純損失（マイナス収支）
- グループ化により、日ごとの勝敗が一目で確認可能。

const LUCK_FACTOR    = 5.0;   // 運要素（大きいほどランダム）

## 🔧 既知の設計事項

- **ハウスエッジなし**: 還元率は理論上100%に近い（ゲームバランス優先）
- **growth_type**: 馬の成長タイプはDB保存済みだが、シミュレーターへの組み込みは検討中
- **course_apt**: 左右回り適性はDB保存済みだが、現コースは常に「右回り」固定のため未使用
